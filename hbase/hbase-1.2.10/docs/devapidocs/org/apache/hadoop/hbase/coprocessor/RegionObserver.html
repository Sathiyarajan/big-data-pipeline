Interface RegionObserver

  • All Superinterfaces:
    Coprocessor
    All Known Implementing Classes:
    AccessController, BaseMasterAndRegionObserver, BaseRegionObserver, ConstraintProcessor, VisibilityController, WriteSinkCoprocessor, ZooKeeperScanPolicyObserver

    @InterfaceAudience.LimitedPrivate(value="Coprocesssor")
@InterfaceStability.Evolving
public interface RegionObserver
extends Coprocessor
    Coprocessors implement this interface to observe and mediate client actions on the region.

      • preCompact

        InternalScanner preCompact(ObserverContext<RegionCoprocessorEnvironment> c,
                         Store store,
                         InternalScanner scanner,
                         ScanType scanType,
                         CompactionRequest request)
                           throws IOException
        Called prior to writing the StoreFiles selected for compaction into a new StoreFile. To override or modify the compaction process, implementing classes have two options:
        • Wrap the provided InternalScanner with a custom implementation that is returned from this method. The custom scanner can then inspect KeyValues from the wrapped scanner, applying its own policy to what gets written.
        • Call ObserverContext.bypass() and provide a custom implementation for writing of new StoreFiles. Note: any implementations bypassing core compaction using this approach must write out new store files themselves or the existing data will no longer be available after compaction.
        Parameters:
        c - the environment provided by the region server
        store - the store being compacted
        scanner - the scanner over existing data used in the store file rewriting
        scanType - type of Scan
        request - the requested compaction
        Returns:
        the scanner to use during compaction. Should not be null unless the implementation is writing new store files on its own.
        Throws:
        IOException - if an error occurred on the coprocessor
