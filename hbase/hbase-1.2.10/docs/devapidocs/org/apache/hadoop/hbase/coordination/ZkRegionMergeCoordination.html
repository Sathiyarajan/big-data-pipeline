                                 RegionMergeCoordination.RegionMergeDetails details)

        Wait for the merging node to be transitioned from pending_merge to merging by master. That's how we are sure master has processed the event and is good with us to move on. If we don't get any update, we periodically transition the node so that master gets the callback. If the node is removed or is not in pending_merge state any more, we abort the merge.

        Creates a new ephemeral node in the PENDING_MERGE state for the merged region. Create it ephemeral in case regionserver dies mid-merge.

        Does not transition nodes from other states. If a node already exists for this region, a KeeperException.NodeExistsException will be thrown.
