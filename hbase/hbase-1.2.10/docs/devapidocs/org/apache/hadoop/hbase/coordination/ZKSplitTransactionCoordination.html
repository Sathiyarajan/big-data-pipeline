      • startSplitTransaction

        public void startSplitTransaction(HRegion parent,
                         ServerName serverName,
                         HRegionInfo hri_a,
                         HRegionInfo hri_b)
                           throws IOException
        Creates a new ephemeral node in the PENDING_SPLIT state for the specified region. Create it ephemeral in case regionserver dies mid-split.

        Does not transition nodes from other states. If a node already exists for this region, an Exception will be thrown.

      • waitForSplitTransaction

        public void waitForSplitTransaction(RegionServerServices services,
                           Region parent,
                           HRegionInfo hri_a,
                           HRegionInfo hri_b,
                           SplitTransactionCoordination.SplitTransactionDetails sptd)
                             throws IOException
        Wait for the splitting node to be transitioned from pending_split to splitting by master. That's how we are sure master has processed the event and is good with us to move on. If we don't get any update, we periodically transition the node so that master gets the callback. If the node is removed or is not in pending_split state any more, we abort the split.
